Microsoft environment equivalents

The Microsoft compilers offer several so-called subsystems, and some choices are made by the compiler based on the application sources offered to it. The CeGCC compiler doesn't offer all of that behaviour, as we outline below.

Microsoft subsystem short decsription CeGCC equivalent ?
CONSOLE character-mode applications not offered (currently, the Voxware tools had something in this area). It is possible to create applications without a console but also without windows (applications that just manipulate files) though.
WINDOWS, WINDOWSCE applications that create their own windows this is the standard behaviour for the arm-wince-mingw32ce tools
NATIVE this applies to device drivers not offered
POSIX unix-like applications this is the standard behaviour for the arm-wince-cegcc tools

Important : the above is interpretation. You may be inclined to believe that it is more recommended to use the arm-wince-mingw32ce tools, or the other way around. We offer both sets of tools, neither is believed to be superior to the other. They both have their uses.

The Microsoft tools have some default behaviour, such as if main is defined then CONSOLE is the default. CeGCC doesn't mimic that behaviour.

The /ENTRY option to the Microsoft compiler is equivalent to CeGCC's -e option.
